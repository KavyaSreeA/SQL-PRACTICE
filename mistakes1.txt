1. INSERT Statement â€” Mistake Patterns & Fixes

Mistake: Missing `INTO` and table name
Wrong:
INSERT CustomerName, City, Country VALUES(...)

Correct:
INSERT INTO Customers (CustomerName, City, Country)
VALUES (...)

Lesson: Always include

* INTO
* table name
* column list inside parentheses

---

Mistake: Inserting values without specifying columns
Wrong:
INSERT INTO Customers VALUES (...)

Risk:

* breaks if table structure changes
* fails if auto-increment exists

Correct (safe):

INSERT INTO Customers
(CustomerName, ContactName, Address, City, PostalCode, Country)
VALUES (...)

Lesson: ALWAYS specify column names in INSERT.

---

Mistake: Mixing SELECT + INSERT
Wrong:
SELECT CustomerName, Country VALUES(...)

Correct:
INSERT INTO Customers (CustomerName, Country)
VALUES (...)

Lesson:
INSERT adds rows â€” SELECT retrieves rows.

---

Mistake: Multi-row insert without column names

Correct:

INSERT INTO Customers
(CustomerName, ContactName, Address, City, PostalCode, Country)
VALUES (...), (...)

Lesson: All rows share same column order.

Auto-Increment Understanding

2. NULL & IS NULL â€” Mistake Patterns & Fixes

Wrong:

Correct:

Wrong:

Mistake: Missing FROM clause
Wrong:

Correct:

Lesson:
SELECT requires:

3. UPDATE Statement â€” Mistake Patterns & Fixes

Mistake: Semicolon before WHERE (dangerous)

Wrong:

SET ContactName = 'Emma';
WHERE CustomerID = 3;

Effect: First statement updated ALL rows

Correct:

UPDATE Customers
SET ContactName = 'Emma'
WHERE CustomerID = 3;

Lesson: Never end UPDATE before WHERE.

---

Mistake: Updating without table name

Wrong:

Correct:
UPDATE Customers
SET CustomerName = ...

---

Mistake: Updating ALL rows accidentally

UPDATE Customers
SET City = 'Rome';

Effect: every record becomes Rome

Lesson: Always confirm WHERE before UPDATE.

Safe Workflow Habit

1. Run SELECT with same WHERE
2. Verify affected rows
3. Then run UPDATE
